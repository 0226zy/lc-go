# 4. 寻找两个正序数组的中位数 (Median of Two Sorted Arrays)

## 题目描述

给定两个大小分别为 `m` 和 `n` 的正序（从小到大）数组 `nums1` 和 `nums2`。请你找出并返回这两个正序数组的 **中位数**。

算法的时间复杂度应该为 `O(log(m+n))`。

### 示例 1

```
输入：nums1 = [1,3], nums2 = [2]
输出：2.00000
解释：合并数组 = [1,2,3]，中位数 2
```

### 示例 2

```
输入：nums1 = [1,2], nums2 = [3,4]
输出：2.50000
解释：合并数组 = [1,2,3,4]，中位数 (2+3)/2 = 2.5
```

## 提示

- `nums1.length == m`
- `nums2.length == n`
- `0 <= m <= 1000`
- `0 <= n <= 1000`
- `1 <= m + n <= 2000`
- `-10^6 <= nums1[i], nums2[i] <= 10^6`

## 题目解析

### 核心思路

这道题的关键在于满足 **O(log(m+n))** 的时间复杂度要求，因此不能简单合并后排序（O(m+n)），而需要使用 **二分查找**。

核心思想是将问题转化为：在两个有序数组中找到一个分割线，使得左半部分的所有元素都小于等于右半部分的所有元素。

1. **二分查找较短数组**：始终对较短的数组做二分，保证时间复杂度为 O(log(min(m,n)))

2. **分割线定义**：在 nums1 中切 i 个元素，在 nums2 中切 j = (m+n+1)/2 - i 个元素，使得左半部分共有 (m+n+1)/2 个元素

3. **合法条件**：`nums1[i-1] <= nums2[j]` 且 `nums2[j-1] <= nums1[i]`，即左边的最大值不超过右边的最小值

4. **中位数计算**：
   - 总长度为奇数：中位数 = 左半部分的最大值
   - 总长度为偶数：中位数 = (左半部分最大值 + 右半部分最小值) / 2

### 算法步骤

1. 确保 nums1 是较短的数组（若不是则交换）
2. 在 nums1 上二分搜索分割位置 `i`（范围 `[0, m]`）
3. 根据 `i` 计算 `j = (m+n+1)/2 - i`
4. 比较边界值判断分割是否合法：
   - `nums1[i-1] > nums2[j]`：i 偏大，右移左边界
   - `nums2[j-1] > nums1[i]`：i 偏小，左移右边界
   - 否则：找到合法分割
5. 根据奇偶性计算中位数

### 复杂度分析

- **时间复杂度**: O(log(min(m, n)))，只对较短数组做二分
- **空间复杂度**: O(1)，只使用常数额外空间
